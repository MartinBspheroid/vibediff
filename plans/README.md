## Findings considered and rejected

- **WebSocket hub data race**: NOT a race — `WSHub.clients` map is only ever
  touched inside the single `Run()` goroutine via channels (`websocket.go:48-89`).
  Safe by design.
- **Review store data race**: NOT a race — `Store` guards `comments` with
  `sync.RWMutex` (`store.go:20-73`). Safe.
- **`diffTarget` command injection** (`service.go:34`): the target comes from a
  CLI positional arg controlled by the operator running the binary, never from
  an HTTP request. `exec.Command` uses arg-slice (no shell). Defense-in-depth
  `--` separator is folded into plan 002 as a minor step, not a standalone bug.
- **Double-close of WebSocket conn on shutdown** (`websocket.go:79`): gorilla's
  `Conn.Close()` returns an error on second call rather than panicking; no crash.
  Cosmetic; not worth a dedicated plan (the redundant-Close can be cleaned up
  opportunistically during plan 005).
- **Virtual scrolling for large diffs** (PERFORMANCE-02): real, but L-effort
  architectural change that only matters for very large (5k+ line) diffs. Left as
  a direction option for the maintainer, not a near-term plan.
